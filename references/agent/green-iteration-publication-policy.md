# Публикация зелёной итерации

Этот контракт регулирует автоматический checkpoint после успешно завершённой итерации в репозитории `qa-agent-v2`.

## Когда итерация считается зелёной

Итерация зелёная, только если:

- запрошенный результат реализован, а обязательная работа текущего scope завершена;
- после последнего содержательного изменения прошли относящиеся к итерации тесты, gates и `git diff --check`;
- нет известного blocking-дефекта в изменённых файлах или обязательном маршруте;
- рабочая ветка соответствует `codex/*`.

Недоступная обязательная проверка, незавершённая реализация или внешний blocker не считаются зелёным результатом. Явно согласованный nonblocking skip допустим только если он не подменяет требуемый gate.

## Commit scope

После зелёной итерации агент действует без дополнительного запроса пользователя:

- определяет точный список созданных и изменённых им файлов этой итерации;
- сверяет `git status` и staged diff, сохраняя все посторонние пользовательские изменения вне индекса;
- добавляет файлы явными pathspec-ами; `git add -A` и эквивалентный захват всего worktree запрещены;
- не добавляет ignored/local inputs, исходные FT-документы, секреты, diagnostics и другие артефакты, исключённые действующей release policy;
- создаёт один отдельный содержательный commit; пустой commit не создаётся.

Если принадлежность файла итерации нельзя доказать, файл не включается. Неоднозначность, из-за которой невозможно собрать целостный commit, является локальным blocker публикации, но не разрешает захватывать чужие изменения.

## Push policy

После успешного commit агент выполняет обычный push текущей `codex/*` ветки в её upstream; при отсутствии upstream использует `origin` и устанавливает tracking.

- Force-push, переписывание remote history и автоматический rebase запрещены.
- Теги и pull request создаются только по отдельному указанию или release-контракту.
- Non-fast-forward, auth failure, branch protection и network failure не отменяют локальный commit: агент сохраняет его, сообщает точную ошибку и команду продолжения.
- Если итерация не изменила отслеживаемые файлы, агент сообщает, что commit и push не требуются.

Это постоянное разрешение распространяется только на зелёные итерации этого репозитория в текущих ветках `codex/*`; оно не разрешает публикацию из других веток или других репозиториев.
