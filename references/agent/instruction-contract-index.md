# Instruction Contract Index

Этот индекс связывает ключевые правила агентного слоя с их каноническими источниками, потребителями и автоматическими проверками.

Он не заменяет `content-placement.md`, `duplication-policy.md`, `skill-boundaries.md` и форматные references. Его задача - показать, где у правила единственный источник истины и чем оно реально проверяется.

## Как пользоваться

- Если нужно изменить правило, сначала найди его строку в этом индексе.
- Меняй канонический источник, а не копии в нескольких `SKILL.md`.
- После изменения правила обнови связанные skill-и только ссылками или краткими напоминаниями.
- Если правило важно для pipeline, добавь или обнови проверку в `scripts/validate_agent_artifacts.py` и тесты.
- Если правило пока не проверяется автоматически, явно оставь `validator coverage = none` или `partial`.

## Ownership Rules

### `skills/*/SKILL.md`

Keep only phase-specific workflow:

- when to use the skill;
- required inputs;
- expected outputs;
- operational steps;
- explicit non-goals;
- links to canonical references.

Do not copy full QA rules, format definitions, or global placement policy into a skill.

### `references/agent/*`

Keep agent-layer contracts:

- process state;
- artifact placement;
- stage handoffs;
- skill boundaries;
- instruction architecture;
- artifact portability.

### `references/qa/*`

Keep QA methodology and QA artifact formats:

- test case structure;
- traceability rules;
- review finding rules;
- coverage checklist;
- UI automation prep format.

### `README.md` and `references/agent/user-manual.md`

Keep navigation and human-facing walkthroughs.

These files should point to canonical sources rather than introduce new authoritative rules.

## Validator Coverage Levels

- `strong` - root validator checks the contract directly and has focused tests.
- `medium` - validator or named suite checks important structural parts, but semantic quality still requires review.
- `partial` - checks exist only in helper scripts or indirect tests and are not exposed through the canonical runner.
- `none` - no automated coverage; rule is manual only.
