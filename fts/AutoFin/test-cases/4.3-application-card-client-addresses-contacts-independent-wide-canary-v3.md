# Тест-кейсы: карточка заявки, адреса и контакты клиента

Набор покрывает видимые UI-обязанности раздела 4.3 по адресам клиента, контактам клиента и контактным лицам. Диагностические материалы v2 находятся в `fts/AutoFin/work/canary-runs/independent-wide-canary-v3/`.

## TC-AF43-AW3-001

**Название:** Поле "Адрес регистрации" показывает найденный адрес через DaData
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-01
**Трассировка:** BSR 115; BSR 116; BSR 119; Table 7 row 33; ATOM-AW3-001
**Сценарное обоснование:** Проверяется стартовое отображение блока «Адрес регистрации» по Table 7 row 33: видимость блока, отсутствие уже заполненного адреса и доступность блока являются одним начальным UI-состоянием. Ограничения ввода адресных компонентов и интеграционные проверки DaData/kladr покрываются отдельными TC или GAP.

**Предусловия:**
1. Открыть карточку заявки, доступную для редактирования.
2. Перейти к блоку "Адреса клиента".
3. Убедиться, что признак ручного ввода адреса регистрации установлен в значение "Нет".

**Тестовые данные:**
- Адрес регистрации: `г Красноярск, ул Мира, д 10`.

**Шаги:**
1. В поле "Адрес регистрации" ввести `г Красноярск, ул Мира, д 10`.
2. Выбрать найденный адрес из подсказок.

**Итоговый ожидаемый результат:**
В поле "Адрес регистрации" отображается выбранный адрес `г Красноярск, ул Мира, д 10`.

**Постусловия:**
Закрыть карточку заявки без сохранения изменений.

## TC-AF43-AW3-003

**Название:** Адрес регистрации без квартиры требует квартиру или признак частного дома
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-01
**Трассировка:** BSR 117; BSR 133; BSR 137; Table 7 rows 33, 43, 44; ATOM-AW3-003
**Сценарное обоснование:** Проверяется единая UI-композиция адреса регистрации после выбора/заполнения адреса: source rows связывают отображение итогового адреса и связанных признаков в одном адресном блоке. Ограничения отдельных компонентов адреса и альтернативные ветки покрываются отдельными TC/GAP.

**Предусловия:**
1. Открыть карточку заявки, доступную для редактирования.
2. Перейти к блоку "Адреса клиента".
3. Выбрать адрес регистрации с регионом и номером дома, но без квартиры.
4. Убедиться, что признак "Клиент проживает в частном доме" установлен в значение "Нет".

**Тестовые данные:**
- Адрес регистрации: `г Красноярск, ул Мира, д 10`.

**Шаги:**
1. Проверить состояние поля "Квартира" для адреса регистрации.

**Итоговый ожидаемый результат:**
Поле "Квартира" подсвечено и отображается подсказка `Укажите номер квартиры или поставьте отметку о проживании в частном доме`.

**Постусловия:**
Закрыть карточку заявки без сохранения изменений.
