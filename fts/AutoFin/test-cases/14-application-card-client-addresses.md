**Тип:** позитивный
**Приоритет:** низкий
**package_id:** WP-01
**Трассировка:** OBL-001; ATOM-018; SRC-018

### Предусловия

1. Открыть анкету клиента.

### Тестовые данные

- Анкета клиента.

### Шаги

1. Перейти к адресной части анкеты клиента.

### Итоговый ожидаемый результат

На странице виден заголовок «Адреса клиента».

## TC-002

**Название:** Видимость поля «Адрес регистрации» в автоматическом и ручном режимах
**Тип:** позитивный
**Приоритет:** средний
**package_id:** WP-01
**Трассировка:** OBL-002; ATOM-019; SRC-019; BSR 115

### Постусловия

- Не применимо: требования не задают восстановление значения переключателя.

## TC-004

**Название:** Обязательность региона в адресе регистрации
**Тип:** негативный
**Приоритет:** высокий
**package_id:** WP-01
**Трассировка:** OBL-004; ATOM-021; SRC-019; BSR 117
**Статус oracle:** ui-calibration-required
**Статус тест-кейса:** candidate-ui-calibration
**Требуется подтверждение:** Какое действие в UI запускает проверку обязательности региона и какой наблюдаемый отклик возникает при пустом регионе?

### Предусловия

1. Открыть анкету клиента.
2. Перейти в блок «Адреса клиента».
3. Открыть заполнение адреса регистрации, позволяющее оставить значение поля «Регион» пустым.
4. Заполнить остальные доступные компоненты адреса регистрации допустимыми значениями, включая номер дома.

### Тестовые данные

- Поле «Регион»: пустое значение.
- Поле номера дома: допустимое непустое значение.

### Шаги

1. Оставить поле «Регион» пустым.
2. Во время UI-калибровки выполнить фактическое действие продолжения или проверки адреса и зафиксировать выполненное действие.

### Итоговый ожидаемый результат

Пустое обязательное поле «Регион» не принимается как валидное для продолжения. Во время UI-калибровки должны быть зафиксированы точное выполненное действие, видимое состояние поля и фактический результат проверки без предположения о конкретном механизме реакции.

### Постусловия

- Не применимо: тест-кейс предназначен для калибровки наблюдаемого UI-отклика.

## TC-014

**Название:** Значение по умолчанию «Нет» переключателя «Ввести вручную» без отдельной активации в адресе регистрации
**Тип:** позитивный
**Приоритет:** средний
**package_id:** WP-01
**Трассировка:** OBL-014; ATOM-031; SRC-020
**Статус oracle:** ui-calibration-required
**Статус тест-кейса:** candidate-ui-calibration
**Требуется подтверждение:** Какой фактический видимый отклик интерфейса возникает при сохранении значения по умолчанию «Нет» без отдельной активации переключателя «Ввести вручную»?

### Предусловия

1. Открыть новую анкету, в которой переключатель «Ввести вручную» ранее не изменялся.
2. Открыть блок адреса регистрации.
3. Просмотреть переключатель «Ввести вручную» и убедиться, что он отображает значение по умолчанию «Нет».

### Тестовые данные

- Логическое значение по умолчанию: «Нет».

### Шаги

1. Не изменять значение «Нет» переключателя «Ввести вручную» и осмотреть его отображаемое состояние.

### Итоговый ожидаемый результат

В записи evidence зафиксированы: выполненное действие — осмотр переключателя без изменения; входное значение — «Нет»; видимое состояние — переключатель отображает «Нет»; фактический видимый отклик интерфейса на выполненное действие. Значение «Нет» допускается без отдельной активации переключателя. Необязательный логический элемент остаётся в заданном значении по умолчанию «Нет» без обязательной активации; точный UI-триггер и итоговый отклик требуют калибровки.

### Постусловия

- Переключатель «Ввести вручную» не изменён и отображает значение «Нет».

## TC-109

**Название:** Раскладка выбранного через DaData адреса фактического места жительства по видимым полям ручного ввода
**Тип:** позитивный
**Приоритет:** высокий
**package_id:** WP-01
**Трассировка:** OBL-109; ATOM-126; SRC-044; BSR 324; CLR-ADDR-005

### Предусловия

1. Открыть блок «Адреса клиента».
2. Установить признак «Адрес фактического места жительства совпадает с адресом регистрации» в значение «Нет».
3. Убедиться, что отображается штатное поле «Адрес фактического места жительства».
4. Установить признак «Ввести вручную» в значение «Нет».

### Тестовые данные

- Адрес фактического места жительства, предложенный штатным запросом DaData во время выполнения теста.

### Шаги

1. В штатном поле «Адрес фактического места жительства» выполнить поиск тестового адреса и выбрать предложенную подсказку DaData.
2. До открытия полей ручного ввода зафиксировать компоненты адреса, возвращённые для выбранной подсказки DaData.
3. Установить признак «Ввести вручную» в значение «Да», чтобы отобразить поля ручного ввода выбранного адреса.
4. Сравнить значение каждого отображаемого поля ручного ввода с соответствующим компонентом, зафиксированным для выбранной подсказки.

### Итоговый ожидаемый результат

Каждое отображаемое поле ручного ввода содержит значение соответствующего компонента, зафиксированного для выбранной подсказки DaData.

### Постусловия

- Не применимо: проверяется только видимая раскладка выбранного адреса по полям ручного ввода.
