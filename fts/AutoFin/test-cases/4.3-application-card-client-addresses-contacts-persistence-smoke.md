# 4.3 Карточка заявки: persistence smoke для адресов и контактов клиента

Набор проверяет минимальную персистентность валидных данных после сохранения и повторного открытия карточки заявки. Точный UI-механизм сохранения в source artifacts для раздела 4.3 не найден; каждый TC помечен как требующий подтверждения save flow перед исполнением. Путь повторного открытия карточки source-backed: `BSR 35`.

## TC-AF43-PS-001

**Название:** Выбранный через DaData адрес регистрации сохраняется после повторного открытия карточки заявки
**Тип:** Positive
**Приоритет:** High
**package_id:** PS-01
**Трассировка:** BSR 35; BSR 115; BSR 116; BSR 119; Table 7 row 33
**Статус oracle:** save-flow-calibration-required
**Статус тест-кейса:** candidate-persistence-calibration
**Требуется подтверждение:** Подтвердить точное действие сохранения карточки заявки и способ выхода из карточки после сохранения.

**Сценарное обоснование:** Проверяется persistence smoke для адреса регистрации: field-level выбор адреса через DaData покрыт v4 canary, здесь проверяется сохранение валидного выбранного значения и его отображение после повторного открытия той же карточки.

**Предусловия:**
1. Открыта карточка заявки, доступная для редактирования.
2. Открыт блок "Адреса клиента".
3. Признак ручного ввода адреса регистрации установлен в значение "Нет".

**Тестовые данные:**
- Адрес регистрации: `г Красноярск, ул Мира, д 10`.

**Шаги:**
1. В поле "Адрес регистрации" ввести `г Красноярск, ул Мира, д 10`.
2. Выбрать найденный адрес из подсказок DaData.
3. Сохранить карточку заявки подтвержденным для раздела 4.3 способом.
4. Закрыть карточку заявки.
5. Повторно открыть ту же карточку заявки через действие перехода в выбранную заявку.
6. Перейти к блоку "Адреса клиента".

**Итоговый ожидаемый результат:**
После повторного открытия карточки в поле "Адрес регистрации" отображается сохраненный адрес `г Красноярск, ул Мира, д 10`.

**Постусловия:**
Вернуть адрес регистрации заявки в исходное состояние подтвержденным способом или использовать изолированную тестовую заявку.
