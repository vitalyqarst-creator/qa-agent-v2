## Scope Boundaries

В scope входят только наблюдаемые элементы карточки заявки, связанные с виджетом и входом в кредитный калькулятор:

- отображение виджета `Краткая информация с калькулятора` в карточке заявки;
- отображение в виджете параметров `Сумма кредита, Р`, `VIN`, `Ставка, %`, `Платеж в месяц, Р`, `Срок, мес.` и их соответствие значениям этапа `Кредитный калькулятор` для той же заявки;
- переход на этап `Кредитный калькулятор` при нажатии на виджет;
- открытие окна `Кредитный калькулятор` при нажатии на кнопку `Кредитный калькулятор`;
- наличие предзаполненных данных в окне `Кредитный калькулятор` сразу после открытия и до пользовательского ввода.

В scope не входят расчеты калькулятора, выбор предложений, проверка всего экрана `Кредитный калькулятор`, полный состав предзаполняемых полей и точный mapping предзаполненных значений к данным заявки. Эти проверки требуют внешнего ФТ `Калькулятор` и зафиксированы как `GAP-001`.

## TC-ACCS-001

**Название:** Отображение виджета `Краткая информация с калькулятора` в карточке заявки
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-01

### Шаги

1. Открыть из списка заявок карточку заявки `FIX-ACCS-ANY-CARD`.

### Итоговый ожидаемый результат

В карточке заявки отображается виджет `Краткая информация с калькулятора`.

### Постусловия

- Не требуются; данные заявки не изменяются.

**UI Verification Status:** confirmed

**UI Evidence:**
- `UIE-ACCS-001-SCREENSHOT-01`
- `UIE-ACCS-001-SCREENSHOT-02`

**Automation Notes:** Использовать локальный restricted setup profile. Downstream automation заблокирована до появления безопасной воспроизводимой fixture; evidence не публикуется.

## TC-ACCS-002

**Название:** Отображение в виджете пяти параметров этапа `Кредитный калькулятор`
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-01
**Трассировка:** OBL-002; ATOM-002; BSR 44; SRC-001

### Предусловия

1. Пользователь авторизован с правом открытия карточек заявок и этапа `Кредитный калькулятор` в заявке.
2. В системе доступна fixture `FIX-ACCS-CALC-SUMMARY-01`: существующая заявка, у которой на этапе `Кредитный калькулятор` отображаются сохраненные значения для параметров `Сумма кредита, Р`, `VIN`, `Ставка, %`, `Платеж в месяц, Р`, `Срок, мес.`.
3. Перед выполнением шагов пользователь открыл этап `Кредитный калькулятор` для заявки `FIX-ACCS-CALC-SUMMARY-01`, зафиксировал фактически отображаемые значения пяти параметров для этой же заявки и вернулся в карточку заявки без ввода новых значений.

### Тестовые данные

- `FIX-ACCS-CALC-SUMMARY-01`: заявка с зафиксированными перед проверкой значениями пяти параметров этапа `Кредитный калькулятор`; конкретные значения берутся из интерфейса этапа для этой заявки и не задаются тестом.

### Шаги

1. Открыть карточку заявки `FIX-ACCS-CALC-SUMMARY-01`.
2. Найти виджет `Краткая информация с калькулятора`.
3. Сопоставить названия и значения параметров в виджете с ранее зафиксированными значениями этапа `Кредитный калькулятор` для этой же заявки.

### Итоговый ожидаемый результат

Виджет отображает параметры `Сумма кредита, Р`, `VIN`, `Ставка, %`, `Платеж в месяц, Р`, `Срок, мес.`; значение каждого параметра совпадает с зафиксированным значением одноименного параметра на этапе `Кредитный калькулятор` для этой же заявки.

### Постусловия

- Не требуются; данные заявки не изменяются.

**UI Verification Status:** blocked-observability

**UI Evidence:**
- `UIE-ACCS-002-SNAPSHOT-01`
- `UIE-ACCS-002-TRACE-01`

**Automation Notes:** На локальной restricted fixture наблюдались пять заполненных summary values, но independent calculator-stage source values недоступны через normal UI path. Не автоматизировать equality assertion до появления воспроизводимого setup/navigation path к calculator stage для той же безопасной fixture.

## TC-ACCS-005

**Название:** Наличие предзаполненных данных при открытии окна `Кредитный калькулятор`
**Тип:** Positive
**Приоритет:** Medium
**package_id:** WP-01
**Трассировка:** OBL-005; ATOM-005; BSR 46; SRC-002; GAP-001

### Предусловия

1. Пользователь авторизован с правом открытия карточек заявок.
2. В системе доступна fixture `FIX-ACCS-PREFILL-01`: существующая карточка заявки с сохраненными данными заявки, которую пользователь может открыть из списка заявок и в которой доступна кнопка `Кредитный калькулятор`.
3. В рамках текущего выполнения теста пользователь еще не открывал окно `Кредитный калькулятор` и не вводил данные в этом окне для заявки `FIX-ACCS-PREFILL-01`.

### Тестовые данные

2. Нажать кнопку `Кредитный калькулятор`.
3. Сразу после открытия окна, до любого пользовательского ввода, проверить видимые элементы ввода и отображения данных в окне `Кредитный калькулятор`.

### Итоговый ожидаемый результат

В окне `Кредитный калькулятор` до пользовательского ввода присутствует хотя бы одно видимое непустое предзаполненное значение в элементе, отображающем данные заявки.

### Постусловия

- Закрыть окно `Кредитный калькулятор` без изменения данных штатным действием закрытия окна.

**UI Verification Status:** blocked-observability

**UI Evidence:**
- `UIE-ACCS-005-SCREENSHOT-01`
- `UIE-ACCS-005-TRACE-01`

**Automation Notes:** Окно calculator не открылось, поэтому prefill до пользовательского ввода не наблюдаем. После восстановления entrypoint повторить проверку наличия хотя бы одного data-bearing значения; не проверять полный состав или exact mapping по `GAP-001`.
