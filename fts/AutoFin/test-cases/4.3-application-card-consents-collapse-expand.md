# Тест-кейсы

## TC-AUTOFIN-CONSENTS-001

**Название:** Исходное свернутое состояние блока «Согласия / проверки» в новой пустой заявке
**Тип:** позитивный
**Приоритет:** средний
**package_id:** WP-01
**Трассировка:** OBL-001; ATOM-001; SRC-007; BSR 317

### Предусловия

1. Открыть пользовательский интерфейс и перейти к доступной кнопке «Создать заявку».

### Тестовые данные

- Новая заявка без введенных данных.

### Шаги

1. Нажать «Создать заявку» и не заполнять открывшуюся карточку заявки.

### Итоговый ожидаемый результат

При первом отображении блок «Согласия / проверки» свернут, область его содержимого не показана.

### Постусловия

- Не применимо: требования к состоянию заявки после проверки не заданы.

## TC-AUTOFIN-CONSENTS-002

**Название:** Раскрытие блока «Согласия / проверки» из свернутого состояния
**Тип:** позитивный
**Приоритет:** средний
**package_id:** WP-01
**Трассировка:** OBL-002; ATOM-002; SRC-007; BSR 317

### Предусловия

1. Открыть пользовательский интерфейс и перейти к доступной кнопке «Создать заявку».
2. Нажать «Создать заявку» и не заполнять открывшуюся карточку заявки; при первом отображении блок «Согласия / проверки» находится в свернутом состоянии, а область его содержимого не показана.

### Тестовые данные

- Новая заявка без введенных данных.

### Шаги

1. Нажать видимый chevron/контрол у заголовка блока «Согласия / проверки».

### Итоговый ожидаемый результат

Блок «Согласия / проверки» раскрыт, область его содержимого видима.

### Постусловия

- Не применимо: требования к состоянию блока после проверки не заданы.
