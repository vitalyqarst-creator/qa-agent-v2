## TC-PREP-001

**Название:** Ограничение одиночного выбора в виджете типа «Список»  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** autofin-widget-selection-prepared-v2  
**Трассировка:** ATOM-PREP-001  
**Источник:** SRC-001

### Предусловия

1. Runtime fixture предоставляет доступный пользователю виджет `WIDGET_SINGLE` типа «Список».
2. В виджете `WIDGET_SINGLE` доступны как минимум два разных значения из связанного с ним справочника.
3. Значения `VALUE_A` и `VALUE_B` определены из фактически доступных значений виджета по правилам раздела «Тестовые данные».

### Тестовые данные

- `WIDGET_SINGLE` — параметр runtime fixture: виджет типа «Список» без предположения о конкретном экране или названии поля.
- `VALUE_A` — любое значение, фактически доступное для выбора в `WIDGET_SINGLE`.
- `VALUE_B` — другое фактически доступное значение в `WIDGET_SINGLE`; `VALUE_B` не равно `VALUE_A`.

### Шаги

1. Открыть перечень значений виджета `WIDGET_SINGLE`.
2. Выбрать `VALUE_A`.
3. Повторно открыть перечень значений виджета `WIDGET_SINGLE`.
4. Выбрать `VALUE_B`.
5. Проверить набор значений, отображаемых в виджете как выбранные.

### Итоговый ожидаемый результат

После последовательного выбора `VALUE_A` и `VALUE_B` виджет содержит ровно одно выбранное значение; два значения не отображаются выбранными одновременно.

### Постусловия

- Не применимо: кейс проверяет только текущее UI-состояние; сохранение, persistence, API и БД не входят в scope.

## TC-PREP-003

**Название:** Начальное отсутствие выбранного значения в новом виджете  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** autofin-widget-selection-prepared-v2  
**Трассировка:** ATOM-PREP-003  
**Источник:** SRC-003

### Предусловия

1. Runtime fixture предоставляет доступный пользователю новый виджет `WIDGET_NEW`, к которому пользователь еще не применял действия выбора.
2. Открыто UI-состояние, в котором `WIDGET_NEW` впервые доступен для просмотра до взаимодействия с ним.

### Тестовые данные

- `WIDGET_NEW` — параметр runtime fixture: новый подходящий виджет без заранее заданного значения и без предположения о конкретном экране или названии поля.

### Шаги

1. Не выполняя действий с `WIDGET_NEW`, проверить отображаемое в нем выбранное значение.

### Итоговый ожидаемый результат

До первого взаимодействия с новым виджетом `WIDGET_NEW` в нем отсутствует выбранное значение.

### Постусловия

- Не применимо: состояние виджета не изменяется; внутренняя интерпретация пустого значения как `NULL` данным кейсом не проверяется.

## Coverage gaps

### GAP-PREP-001

**Статус:** non-blocking, executable TC не создается  
**Трассировка:** ATOM-PREP-004  
**Источник:** SRC-003

Внутренняя интерпретация отсутствующего значения как `NULL` не имеет наблюдаемого артефакта в подтвержденном UI scope. Для проверки требуется отдельный подтвержденный scope с наблюдаемым API-, DB- или persistence-оракулом.

## Неисполняемые границы покрытия

- Негативные проверки не создаются: выбранные требования не задают классы невалидных значений и наблюдаемые негативные оракулы.
- Проверки обязательности не создаются: выбранные требования не определяют requiredness, маркер обязательности или реакцию на пустое значение.
