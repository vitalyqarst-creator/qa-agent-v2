# Тест-кейсы

## TC-ACPD-001

**Название:** Отображение блока «Персональные данные» и полей ФИО
**Тип:** позитивный
**Приоритет:** средний
**package_id:** application-card-client-personal-data-v6
**Трассировка:** OBL-001; ATOM-001; SRC-001; SRC-001.P01; OBL-002; ATOM-002; SRC-002; SRC-002.P01; OBL-009; ATOM-007; SRC-003; SRC-003.P01; OBL-016; ATOM-012; SRC-004; SRC-004.P01

### Предусловия

1. Открыта новая карточка заявки, созданная действием создания заявки из BSR 38.

### Тестовые данные

- Не требуются.

### Шаги

1. Открыть карточку заявки и перейти к разделу с персональными данными.

### Итоговый ожидаемый результат

В карточке заявки отображается блок `Персональные данные` с полями `Фамилия`, `Имя` и `Отчество`.

### Постусловия

- Не требуются.

## TC-ACPD-012

**Название:** Обновление поля «Пол» после выбора подсказки ФИО
**Тип:** негативный
**Приоритет:** средний
**package_id:** application-card-client-personal-data-v6
**Трассировка:** OBL-027; ATOM-020; SRC-006; SRC-006.P02
**Coverage gap:** GAP-003
**Статус oracle:** ui-calibration-required
**Статус тест-кейса:** candidate-ui-calibration

### Предусловия

1. Открыта новая карточка заявки, созданная действием создания заявки из BSR 38.
2. Интеграция DaData доступна.

### Тестовые данные

- `FIX-ACPD-RUNTIME-DADATA-001`: префикс `Иван`; фактически возвращённая подсказка с доступными составными частями ФИО и известным непустым полом.

### Шаги

1. Записать исходное видимое значение поля `Пол`.
2. Ввести `Иван` в поле ФИО, выбрать фактически возвращённую подсказку с известным полом и записать итоговое видимое значение поля `Пол`.

### Итоговый ожидаемый результат

После выбора подсказки видимое значение поля `Пол` совпадает со значением пола выбранной подсказки; исходное и итоговое значения зафиксированы.

### Постусловия

- Не требуются.
