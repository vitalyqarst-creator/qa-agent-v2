## TC-PREP-001

**Название:** Ограничение одиночного выбора в виджете типа «Список»  
**Статус:** unsigned  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** `autofin-widget-selection-prepared-v2`  
**Трассировка:** `ATOM-PREP-001`  
**Источник:** `SRC-001`

### Предусловия

1. Исполнителю доступен runtime fixture с виджетом типа «Список»; точный путь к экрану или форме и идентификатор виджета заданы параметром `fixture_locator`.
2. Виджет доступен для выбора значений.
3. В связанном с виджетом справочнике существуют и доступны для выбора два разных значения, переданные параметрами `value_a` и `value_b`.

### Тестовые данные

- `fixture_locator` — заданный в runtime fixture точный путь к экрану или форме и идентификатор проверяемого виджета типа «Список».
- `value_a` — первое доступное значение из фактически связанного с виджетом справочника.
- `value_b` — второе доступное значение из того же справочника, отличное от `value_a`.

### Шаги

1. Открыть экран или форму и виджет, указанные в `fixture_locator`.
2. Выбрать в виджете значение `value_a`.
3. Повторно открыть выбор значений и выбрать `value_b`.

### Итоговый ожидаемый результат

После последовательного выбора `value_a` и `value_b` виджет содержит ровно одно выбранное значение.

### Постусловия

- Специальные постусловия не применимы: сохранение и проверка persistence не входят в scope; состояние runtime fixture может быть сброшено штатным механизмом тестового окружения.

3. В связанном с виджетом справочнике существуют и доступны для выбора два разных значения, переданные параметрами `value_a` и `value_b`.

## TC-PREP-003

**Название:** Отсутствие выбранного значения в начальном состоянии виджета  
**Статус:** unsigned  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** `autofin-widget-selection-prepared-v2`  
**Трассировка:** `ATOM-PREP-003`  
**Источник:** `SRC-003`

### Предусловия

1. Для каждого набора тестовых данных доступен runtime fixture с новым или сброшенным в исходное состояние виджетом соответствующего типа.
2. До проверки пользователь не взаимодействовал с виджетом и не выбирал в нём значения.
3. Точный путь к экрану или форме и идентификатор виджета заданы соответствующим параметром `fixture_locator`.

### Тестовые данные

Проверку выполнить отдельно для каждого набора:

| Набор | `widget_type` | `fixture_locator` |
| --- | --- | --- |
| `TD-01` | `Список` | Точный путь к экрану или форме и идентификатор нового или сброшенного виджета типа «Список», заданные runtime fixture |
| `TD-02` | `Список с множественным выбором` | Точный путь к экрану или форме и идентификатор нового или сброшенного виджета типа «Список с множественным выбором», заданные runtime fixture |

### Шаги

1. Открыть экран или форму, указанные в `fixture_locator` текущего набора данных.
2. До первого взаимодействия с виджетом проверить его отображаемое значение.

### Итоговый ожидаемый результат

В каждом наборе данных до первого взаимодействия в виджете отсутствует выбранное значение.

### Постусловия

- Не применимы: проверка не изменяет состояние виджета.

- **Трассировка:** `ATOM-PREP-004`, `SRC-003`.
- **Статус:** gap; исполнимый UI-тест не создаётся.
- **Причина:** внутренняя интерпретация отсутствующего значения как `NULL` не имеет наблюдаемого артефакта в подтверждённом UI scope; save-flow, API, БД и persistence исключены из scope.
- **Обработка:** запросить отдельный API/DB/persistence scope, если требуется проверить внутреннее представление значения.
