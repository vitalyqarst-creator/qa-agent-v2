## Параметры runtime fixture

- `W_SINGLE` — доступный в тестовой среде новый виджет типа «Список»; конкретное поле и экран задаются владельцем тестовой среды.
- `W_MULTI` — доступный в тестовой среде новый виджет типа «Список с множественным выбором»; конкретное поле и экран задаются владельцем тестовой среды.
- `W_EMPTY` — доступный в тестовой среде новый виджет одного из типов выбранного scope, с которым после создания ещё не взаимодействовали.
- `V1` и `V2` — два разных значения, фактически доступных для выбора в проверяемом виджете. Значения фиксируются исполнителем из runtime fixture до начала теста; состав и происхождение справочника этим набором не проверяются.

## TC-PREP-001

**Название:** Ограничение одиночного выбора в виджете типа «Список»  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** `autofin-widget-selection-prepared-v3`  
**Трассировка:** `ATOM-PREP-001` (`SRC-001`)

### Предусловия

1. Тестовая среда доступна исполнителю.
2. В среде подготовлен новый runtime fixture с виджетом `W_SINGLE`.
3. В `W_SINGLE` доступны как минимум два разных значения; до выполнения шагов исполнитель зафиксировал их как `V1` и `V2`.

### Тестовые данные

- `W_SINGLE`: виджет типа «Список» из runtime fixture.
- `V1`: первое фактически доступное значение `W_SINGLE`.
- `V2`: второе фактически доступное значение `W_SINGLE`, отличное от `V1`.

### Шаги

1. Открыть подготовленный runtime fixture с `W_SINGLE`.
2. Выбрать в `W_SINGLE` значение `V1`.
3. Выбрать в `W_SINGLE` значение `V2`.
4. Проверить количество и состав значений, отображаемых в `W_SINGLE` как выбранные.

### Итоговый ожидаемый результат

После последовательного выбора `V1` и `V2` виджет `W_SINGLE` содержит ровно одно выбранное значение; одновременно `V1` и `V2` выбранными не отображаются.

### Постусловия

- Очистка или сохранение данных не выполняются: соответствующие потоки не входят в подтверждённый scope.

### Постусловия

- Не применимо: тест не изменяет состояние `W_EMPTY`.

## Coverage gaps

### GAP-PREP-001 — происхождение значений из справочника

- **Трассировка:** `ATOM-PREP-004`, `ATOM-PREP-005`; `SRC-001`, `SRC-002`.
- **Статус:** non-blocking gap; исполнимый тест-кейс не создаётся.
- **Причина:** scope не идентифицирует конкретные справочники и не содержит независимого inventory oracle для проверки состава и происхождения значений.
- **Дальнейшая обработка:** добавить `DICT-*` inventory и отдельные source-origin test cases после подтверждения справочника и ожидаемого состава.

### GAP-PREP-002 — внутренняя интерпретация пустого значения как NULL

- **Трассировка:** `ATOM-PREP-006`; `SRC-003`.
- **Статус:** non-blocking gap; исполнимый UI-тест-кейс не создаётся.
- **Причина:** в подтверждённом UI scope отсутствует наблюдаемый API-, DB- или иной persistence-артефакт внутреннего значения `NULL`.
- **Дальнейшая обработка:** запросить отдельный API/DB/persistence scope, если требуется проверка внутреннего представления.

## Неприменимые проверки

- Негативные проверки не создаются: выбранные строки не определяют классы недопустимых значений и наблюдаемые негативные оракулы.
- Проверки обязательности не создаются: выбранные строки не задают обязательность виджетов.
