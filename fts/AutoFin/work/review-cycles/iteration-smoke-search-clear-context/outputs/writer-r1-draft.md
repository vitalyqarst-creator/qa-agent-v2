# Тест-кейсы: меню «Заявки в системе» - очистка контекста поиска

## Summary

Набор покрывает `BSR 32` из раздела `4.2. Меню «Заявки в системе»`: нажатие кнопки `«Очистить»` очищает фильтры, сортировки, постраничность и состояние выделения строк.

Источник не задает конкретные поля фильтров, значения сортировки, размер страницы, количество строк, сообщения, backend-state, persistence или API-эффекты. Поэтому проверки используют наблюдаемое начальное/default UI state, зафиксированное после открытия раздела, без подстановки неподтвержденных значений.

## TC-BSR32-001

**Название:** Сброс примененного фильтра кнопкой «Очистить»
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-BSR32
**Трассировка:** ATOM-001; BSR 32; SRC-001; section 4.2; PDF page 8

### Предусловия

1. Открыть раздел `Заявки в системе`.
2. Зафиксировать начальное/default состояние элементов фильтрации, видимое сразу после открытия раздела.
3. В одном доступном фильтре раздела установить значение, отличающееся от зафиксированного начального/default состояния.
4. Убедиться, что выбранный фильтр отображает измененное значение.

### Тестовые данные

- Контрольный фильтр: один видимый фильтр раздела, для которого в текущей тестовой фикстуре можно установить значение, отличающееся от начального/default состояния.
- Контрольное значение фильтра: значение, доступное в выбранном фильтре и отличающееся от начального/default состояния.

### Шаги

1. Нажать кнопку `Очистить`.

### Итоговый ожидаемый результат

Контрольный фильтр возвращен в зафиксированное начальное/default состояние; измененное контрольное значение больше не отображается как примененное значение фильтра.

### Постусловия

Не требуются.
