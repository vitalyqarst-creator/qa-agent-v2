## TC-PREP-001

**Название:** Ограничение одиночного выбора в виджете типа «Список»  
**Тип:** позитивный  
**Приоритет:** средний  
**Статус:** unsigned draft  
**package_id:** autofin-widget-selection-prepared-v3  
**Трассировка:** ATOM-PREP-001; SRC-001  
**Runtime fixture:** single_select_widget

### Предусловия

1. Runtime fixture `single_select_widget` предоставляет доступный пользователю UI-контекст с виджетом, соответствующим типу «Список» из `SRC-001`.
2. В виджете доступны для выбора как минимум два разных значения.
3. Фактический UI-контекст и два выбранных значения зафиксированы в протоколе выполнения как параметры `first_value` и `second_value`.

### Тестовые данные

- `first_value` — первое из двух разных значений, фактически доступных в `single_select_widget`.
- `second_value` — второе фактически доступное значение; оно отлично от `first_value`.

### Шаги

1. Открыть UI-контекст, заданный runtime fixture `single_select_widget`.
2. Выбрать в виджете значение `first_value`.
3. В том же виджете выбрать значение `second_value`.
4. Проверить состав значений, отображаемых виджетом как выбранные.

### Итоговый ожидаемый результат

После последовательного выбора `first_value` и `second_value` виджет содержит ровно одно выбранное значение.

### Постусловия

- Требования к сохранённому состоянию неприменимы: сохранение и persistence не входят в подтверждённый scope; результат фиксируется в текущем UI-сеансе.

## TC-PREP-003

**Название:** Начальное пустое состояние нового виджета  
**Тип:** позитивный  
**Приоритет:** средний  
**Статус:** unsigned draft  
**package_id:** autofin-widget-selection-prepared-v3  
**Трассировка:** ATOM-PREP-003; SRC-003  
**Runtime fixture:** new_widget

### Предусловия

1. Runtime fixture `new_widget` предоставляет доступный пользователю UI-контекст с новым виджетом, к которому применяется правило начального состояния из `SRC-003`.
2. Для текущего экземпляра виджета ещё не выполнялись выбор, ввод или иное изменение значения.
3. Фактический UI-контекст и проверяемый виджет зафиксированы в протоколе выполнения.

### Тестовые данные

- `new_widget` — предоставленный runtime fixture новый экземпляр подходящего виджета до первого взаимодействия с его значением.

### Шаги

1. Открыть UI-контекст, заданный runtime fixture `new_widget`, не взаимодействуя со значением виджета.
2. Проверить, отображается ли в виджете выбранное значение.

### Итоговый ожидаемый результат

До первого взаимодействия с новым виджетом в нём отсутствует выбранное значение.

### Постусловия

- Состояние виджета не изменено; проверка внутренней интерпретации значения как `NULL` не выполняется.
