## Параметры runtime-фикстуры

- `single_select_widget` — виджет типа «Список» в поддерживаемом системой контексте создания нового объекта.
- `multi_select_widget` — виджет типа «Список с множественным выбором» в поддерживаемом системой контексте создания нового объекта.
- `empty_state_widget` — новый виджет одного из типов выбранного scope, с которым пользователь еще не взаимодействовал.
- `value_a` и `value_b` — два разных значения, доступных для выбора в проверяемом виджете; конкретные значения предоставляет runtime-фикстура.

Конкретные экран, поля и значения справочников не фиксируются, поскольку они не определены выбранным scope.

## TC-PREP-001

**Название:** Ограничение одиночного выбора в виджете типа «Список»  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** `autofin-widget-selection-prepared-v3`  
**Трассировка:** `ATOM-PREP-001`  
**source_ref:** `SRC-001`

### Предусловия

1. Открыт поддерживаемый системой контекст создания нового объекта с доступным для взаимодействия `single_select_widget`.
2. Для `single_select_widget` runtime-фикстура предоставляет доступные для выбора разные значения `value_a` и `value_b`.
3. До начала проверки в `single_select_widget` отсутствует выбранное значение.

### Тестовые данные

- `value_a` — первое доступное значение runtime-фикстуры.
- `value_b` — второе доступное значение runtime-фикстуры, отличное от `value_a`.

### Шаги

1. Выбрать `value_a` в `single_select_widget`.
2. Выбрать `value_b` в том же `single_select_widget`.
3. Проверить набор значений, отображаемых выбранными в `single_select_widget`.

### Итоговый ожидаемый результат

После последовательного выбора `value_a` и `value_b` виджет содержит ровно одно выбранное значение — `value_b`; `value_a` больше не отображается выбранным.

### Постусловия

- Новый объект не сохраняется; проверка persistence не выполняется.

## TC-PREP-003

**Название:** Начальное пустое состояние нового виджета  
**Тип:** позитивный  
**Приоритет:** средний  
**package_id:** `autofin-widget-selection-prepared-v3`  
**Трассировка:** `ATOM-PREP-003`  
**source_ref:** `SRC-003`

### Предусловия

1. Runtime-фикстура позволяет открыть поддерживаемый системой контекст создания нового объекта с `empty_state_widget`.
2. Новый объект еще не создавался в текущем контексте, а пользователь не взаимодействовал с `empty_state_widget`.

### Тестовые данные

- `empty_state_widget` — новый виджет одного из типов выбранного scope; конкретный виджет предоставляет runtime-фикстура.

### Шаги

1. Открыть контекст создания нового объекта.
2. Не взаимодействуя с `empty_state_widget`, проверить его отображаемое значение.

### Итоговый ожидаемый результат

До первого взаимодействия с новым `empty_state_widget` в нем отсутствует выбранное значение.

### Постусловия

- Новый объект не сохраняется; внутренняя интерпретация пустого значения как `NULL` не проверяется.
