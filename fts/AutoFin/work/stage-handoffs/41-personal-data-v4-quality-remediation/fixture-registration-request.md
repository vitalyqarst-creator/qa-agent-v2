# Запрос на регистрацию fixture для Personal Data V5

## Назначение

Получить два воспроизводимых, безопасных и переносимых входных набора. Результат должен соответствовать `references/agent/fixture-catalog-format.md` и быть достаточным для ручного исполнения TC без восстановления данных из истории чата.

## FIX-ACPD-SAVE-001

Нужно зафиксировать:

| поле | обязательное содержание |
| --- | --- |
| `purpose` | успешное сохранение/переоткрытие тестовой заявки для проверок ID, обязательности и optional persistence |
| `source_ref` | ссылки на `OBL-003`, `010`, `017`, `028`, `029`, `048`, `056`, `064` и соответствующие `SRC-*`/GSR из H41 compiler inputs |
| `setup_state` | точная точка входа, тип тестовой заявки, допустимый статус, права роли и условие resettable/disposable |
| `concrete_data` | полный минимальный набор полей/значений, необходимый для успешного control-save; допускаются ссылки только на другие полностью раскрытые fixtures |
| `dependencies` | нужные справочники/интеграции и подтверждённая доступность на стенде |
| `cleanup` | воспроизводимый reset, discard или удаление тестовой заявки без влияния на общие данные |
| `evidence` | дата/среда проверки и санитарно-безопасное подтверждение, что control-save и cleanup выполнены |

Запрещены формулировки `валидная заявка`, `остальные обязательные поля заполнены`, `значения, необходимые для сохранения` без раскрытия значений.

## FIX-ACPD-DADATA-001

Нужно зафиксировать:

| поле | обязательное содержание |
| --- | --- |
| `purpose` | воспроизводимая проверка автозаполнения поля `Пол` после выбора подсказки ФИО |
| `source_ref` | `OBL-027`, `TC-ACPD-012`, `SRC-006`, `DICT-001` и соответствующий GSR |
| `setup_state` | точная точка входа и подтверждённая доступность DaData на тестовом стенде |
| `concrete_data` | синтетическое вводимое ФИО, точный текст выбираемой подсказки и точное ожидаемое значение `Пол` из `DICT-001` |
| `dependencies` | среда/версия или иная стабильная привязка, при которой suggestion подтверждён |
| `cleanup` | discard/reset несохранённых изменений |
| `evidence` | дата/среда проверки и безопасная фиксация фактической тройки `input → suggestion → Пол` |

Запрещены условные oracle вида `Пол=Мужчина, если подсказка мужская`, а также реальные ФИО клиентов.

## Политика безопасности

- Использовать только синтетические, специально заведённые тестовые или явно разрешённые данные.
- Не коммитить PII, логины, пароли, cookie, access/refresh token, session storage, сетевые trace с секретами и реальные идентификаторы клиентов.
- Если точные значения нельзя безопасно опубликовать, сохранить restricted evidence локально и остановить workflow как `blocked-input`: один opaque alias без раскрытого безопасного execution contract недостаточен для writer-а.

## Gate регистрации

Регистрация завершена только если независимый исполнитель по опубликованному catalog может:

1. подготовить обе fixtures без устной подсказки;
2. выполнить control-save и cleanup для `FIX-ACPD-SAVE-001`;
3. получить точную зарегистрированную тройку `ФИО → suggestion → Пол` для `FIX-ACPD-DADATA-001`;
4. подтвердить отсутствие секретов и PII в публикуемых артефактах.
