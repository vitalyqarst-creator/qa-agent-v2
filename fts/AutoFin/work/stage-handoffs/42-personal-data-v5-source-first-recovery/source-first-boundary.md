# Source-first fixture boundary

## Что достаточно для writer/reviewer iteration

- точная трассировка к ATOM/OBL и строке ФТ;
- действие пользователя, заданное ФТ или явно отмеченное как UI-calibration candidate;
- переносимый класс входных данных: конкретное синтетическое значение, относительная дата либо runtime-selected ответ интеграции с проверяемыми свойствами;
- наблюдаемый UI-oracle из ФТ;
- сохранённый `GAP-*`, если точный механизм/сообщение/реакция не определены.

## Что не является обязательным до UI-prep

- существующий ID заявки на конкретном стенде;
- CSS/XPath/role locator;
- токен, session cookie или опубликованный сетевой trace;
- заранее записанная конкретная подсказка DaData;
- стабильный порядок или полный список динамических подсказок.

## Реальный `blocked-input`

Writer блокируется, только если source evidence не позволяет определить проверяемое намерение или наблюдаемый expected result без выдумывания. Невозможность заранее связать переносимые данные с конкретным стендом фиксируется как UI-prep dependency, но не удаляет FT-first coverage.
