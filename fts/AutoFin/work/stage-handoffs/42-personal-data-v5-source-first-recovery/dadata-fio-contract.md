# Переносимый контракт подсказки ФИО

## Product requirement boundary

- BSR 57: после ввода ФИО пользователь выбирает подсказку DaData, после чего поля ФИО заполняются выбранными значениями.
- BSR 58: после выбора подсказки поле «Пол» обновляется значением, полученным из DaData.
- ФТ не фиксирует конкретное ФИО, порядок подсказок, полный список ответов или внутренний transport contract.

## Runtime-selected fixture

При исполнении в поле ФИО вводится синтетический префикс, который не содержит данные реального клиента. Из фактически возвращённых подсказок выбирается вариант, у которого доступны составные части ФИО и известное непустое значение пола. Выбранные отображаемые значения и итоговое значение поля «Пол» сохраняются в UI evidence.

Если подходящей подсказки нет или интеграция недоступна, это execution blocker/UI-prep evidence, а не основание удалять либо блокировать FT-first test intent.

## External support note

Публичная документация DaData подтверждает форму ответа Suggest FIO (`surname`, `name`, `patronymic`, `gender`) и динамический характер подсказок. Она используется только как support contract; источником бизнес-правил остаётся основное ФТ.
