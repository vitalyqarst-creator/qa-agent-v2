# Итоги V5

## Получилось

- Агент больше не зависит от того, запомнит ли writer две точные fixture из prose prompt.
- Exact значения стали структурной частью package v8, автоматически попадают в draft и проверяются до reviewer.
- Реальный canary принят: 13/13 обязательств, 0 ошибок review.
- Полные справочники по-прежнему не раздувают writer output и добавляются runner-ом один раз.
- Старые packages v1-v7 остаются совместимыми; production test cases не менялись.

## Не получилось доказать

- Один запуск не подтверждает стабильную скорость.
- Расход uncached tokens всё ещё выше V3 примерно на 31% на одно обязательство.
- Решение ещё не проверено на новом, не использованном для настройки scope.

## Практический смысл

Конкретная потеря тестовых данных, из-за которой V4 был забракован, устранена на уровне процесса, а не ручной правкой кейса. Следующий разумный шаг — transfer-canary на другом scope и оптимизация повторного context payload, а не ещё один повтор того же benchmark.
