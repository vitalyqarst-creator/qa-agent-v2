# Итог итерации V4

## Результат простыми словами

Агент действительно перестал дважды печатать огромный справочник. Writer стал короче и в этом запуске быстрее примерно на 27%. Но одновременно он сделал один тест-кейс не полностью исполнимым: вместо двух заданных значений написал «выберите два обычных значения».

Reviewer нашёл эту ошибку, и защитный протокол сработал: набор не принят, production-файлы не изменены, следующие два запуска не потрачены. Поэтому текущую оптимизацию нельзя считать завершённой.

## Процессный вывод

Нельзя полагаться только на то, что модель скопирует конкретные fixture values из prose `test_intent`. Следующая версия должна передавать такие значения структурно и либо заранее помещать их в draft seed, либо блокировать draft до дорогого reviewer-вызова. Это улучшение агента в целом, а не ручная корректировка TC-VAMB-012.
