# Prepared Pipeline Medium-Scope Benchmark

## Цель Этапа

Выбрать один новый independent AutoFin scope с оценкой `8–12` planned TC и `12–25` atomic obligations, чтобы проверить quality, latency и токены обычного prepared writer-reviewer process на medium scope.

## Входные Артефакты

- `fts/AutoFin/AGENT-NOTES.md`
- `fts/AutoFin/work/stage-handoffs/20-prepared-autofin-cross-scope/source-selection.md`
- `fts/AutoFin/work/stage-handoffs/51-search-clear-context-exec-benchmark-v4/live-result.v4.json`
- `fts/AutoFin/work/stage-handoffs/51-search-clear-context-exec-benchmark-v4/performance-analysis.v4.md`
- `fts/AutoFin/work/stage-handoffs/51-search-clear-context-exec-benchmark-v4/iteration-summary.md`
- `fts/AutoFin/work/stage-handoffs/51-search-clear-context-exec-benchmark-v4/stop-gate.md`

## Обязательные Действия

1. Через `ft-scope-analyzer` выбрать один medium scope с mandatory XHTML, DOCX source-of-truth, PDF parity и актуальными source/scope artifacts.
2. Не выбирать H47 employment scope, пока не разрешены его driver-field alias и dictionary divergence.
3. Не использовать user-owned untracked `4.3-application-card-client-addresses-contacts.md` как requirement evidence и не изменять его.
4. Подтвердить scope contract, source parity, row inventory, atomic obligations, dictionaries и gaps; blocking ambiguity останавливает работу до LLM.
5. Выбрать scope с `8–12` planned TC, чтобы первый medium benchmark не смешивался с проверкой sharding.
6. Создать новые numbered handoff, immutable package/cycle и standard structured writer-reviewer config.
7. До live потребовать compile, runtime identity, oracle/state/dictionary gates, context/output capacities, artifact validator, exec dry-run, checkpoint push и separate authorization.
8. После одного authorised dispatcher сравнить duration, total tokens и uncached tokens/obligation с V4.

## Performance Гипотеза

- Target duration: `<120 s` для writer+reviewer, без снижения quality gates.
- Target uncached cost: `<8,000 tokens / obligation`; это measurement goal, а не основание урезать evidence.
- Если цель не достигнута, локализовать fixed writer/reviewer overhead по prompt/instruction/system attribution, а не повторять run.

## Ограничения

- Не возобновлять V4 и не делать promotion.
- Не смешивать scope-ы, SDK fallback и UI automation.
- Не переписывать production test cases в benchmark iteration.

## Gate Завершения

Scope готов к iteration только после current-source/parity/atomic/dictionary/gap closure и явной оценки TC/OBL size. Live benchmark выполняется отдельной iteration и всегда имеет invocation budget `1`.
