# Проверка отрицательных DaData fixtures

## Статус

`blocked-verification`: официальный адресный контракт требует API-ключ; публичная
документация не содержит стабильного примера с ответом `suggestions=[]`.

## Что требуется один раз выполнить перед writer

Для каждого кандидата отправить точный `POST` в официальный endpoint, сохранить
полное тело ответа без токена и убедиться, что ответ равен `suggestions=[]`.
После этого вычислить SHA-256 снимка и создать:

- `FX-DADATA-ADDR-NEG-001.response.json` для `TC-009`;
- `FX-DADATA-ADDR-NEG-002.response.json` для `TC-065` либо явно переиспользовать
  первый fixture, если оба TC действительно проверяют один и тот же продуктовый
  oracle и не должны быть объединены.

До появления такого evidence TC-009 и TC-065 нельзя считать воспроизводимыми и
нельзя передавать writer как production-ready obligations.
