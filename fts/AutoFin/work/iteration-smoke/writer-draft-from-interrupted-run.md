# 3 Ограничения - выбор значений в виджетах типа список

## TC-WIDGET-SELECTION-TYPES-001

**Название:** Выбор нового значения в виджете типа `Список` заменяет ранее выбранное значение
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-01
**Трассировка:** ATOM-001; ATOM-002; SRC-001
**Статус oracle:** ui-calibration-required
**Статус тест-кейса:** candidate-ui-calibration
**Требуется подтверждение:** Конкретный экран, виджет типа `Список` и два активных справочных значения должны быть зафиксированы при UI calibration до перевода кейса в executable baseline.

### Предусловия

1. Открыт экран UI calibration, на котором доступен виджет типа `Список`, соответствующий ограничению `SRC-001`.
2. Для виджета зафиксировано отображаемое название или технический идентификатор `SINGLE_LIST_FIELD`.
3. Для связанного справочника зафиксированы два разных активных значения `SINGLE_VALUE_A` и `SINGLE_VALUE_B`.
4. В `SINGLE_LIST_FIELD` нет выбранного значения.

### Тестовые данные

- `SINGLE_LIST_FIELD` - конкретный виджет типа `Список`.
- `SINGLE_VALUE_A` - первое активное значение справочника для `SINGLE_LIST_FIELD`.
- `SINGLE_VALUE_B` - второе активное значение того же справочника для `SINGLE_LIST_FIELD`.

### Шаги

1. Открыть список значений в `SINGLE_LIST_FIELD`.
2. Выбрать значение `SINGLE_VALUE_A`.
3. Повторно открыть список значений в `SINGLE_LIST_FIELD`.
4. Выбрать значение `SINGLE_VALUE_B`.

### Итоговый ожидаемый результат

В `SINGLE_LIST_FIELD` отображается выбранным только `SINGLE_VALUE_B`; `SINGLE_VALUE_A` не отображается как выбранное значение.

### Постусловия

Не требуются: кейс не сохраняет изменения.

## TC-WIDGET-SELECTION-TYPES-002

**Название:** Виджет типа `Список с множественным выбором` позволяет выбрать несколько справочных значений одновременно
**Тип:** Positive
**Приоритет:** High
**package_id:** WP-01
**Трассировка:** ATOM-003; ATOM-004; SRC-002
**Статус oracle:** ui-calibration-required
**Статус тест-кейса:** candidate-ui-calibration
**Требуется подтверждение:** Конкретный экран, виджет типа `Список с множественным выбором` и два активных справочных значения должны быть зафиксированы при UI calibration до перевода кейса в executable baseline.

### Предусловия

1. Открыт экран UI calibration, на котором доступен виджет типа `Список с множественным выбором`, соответствующий ограничению `SRC-002`.
2. Для виджета зафиксировано отображаемое название или технический идентификатор `MULTI_LIST_FIELD`.
3. Для связанного справочника зафиксированы два разных активных значения `MULTI_VALUE_A` и `MULTI_VALUE_B`.
4. В `MULTI_LIST_FIELD` нет выбранных значений.

### Тестовые данные

- `MULTI_LIST_FIELD` - конкретный виджет типа `Список с множественным выбором`.
- `MULTI_VALUE_A` - первое активное значение справочника для `MULTI_LIST_FIELD`.
- `MULTI_VALUE_B` - второе активное значение того же справочника для `MULTI_LIST_FIELD`.

### Шаги

1. Открыть список значений в `MULTI_LIST_FIELD`.
2. Выбрать значение `MULTI_VALUE_A`.
3. Не очищая `MULTI_VALUE_A`, выбрать значение `MULTI_VALUE_B`.

### Итоговый ожидаемый результат

В `MULTI_LIST_FIELD` одновременно отображаются выбранными `MULTI_VALUE_A` и `MULTI_VALUE_B`.

### Постусловия

Не требуются: кейс не сохраняет изменения.

## TC-WIDGET-SELECTION-TYPES-003

**Название:** Новый виджет отображается без выбранного значения до действий пользователя
**Тип:** Positive
**Приоритет:** Medium
**package_id:** WP-01
**Трассировка:** ATOM-005; SRC-003
**Статус oracle:** ui-calibration-required
**Статус тест-кейса:** candidate-ui-calibration
**Требуется подтверждение:** Конкретный экран и новый виджет без предварительно заданного значения должны быть зафиксированы при UI calibration; внутренняя интерпретация как `NULL` вынесена в `GAP-001`.

### Предусловия

1. Открыт экран UI calibration в состоянии создания новой сущности или нового блока, где доступен целевой виджет `EMPTY_WIDGET_FIELD`.
2. Для `EMPTY_WIDGET_FIELD` зафиксировано отображаемое название или технический идентификатор.
3. Пользователь еще не выбирал и не вводил значение в `EMPTY_WIDGET_FIELD`.

### Тестовые данные

- `EMPTY_WIDGET_FIELD` - конкретный виджет, для которого проверяется отсутствие значения по умолчанию.

### Шаги

1. Просмотреть состояние `EMPTY_WIDGET_FIELD` до выбора или ввода значения.

### Итоговый ожидаемый результат

В `EMPTY_WIDGET_FIELD` не отображается выбранное или заполненное значение.

### Постусловия

Не требуются.
