# Package Notes: AutoFin

## Сокращения В Таблицах UI-Полей

Статус: package-specific рабочее правило для AutoFin, добавленное по подтверждению пользователя и сверенное с аналогичными заметками в `fts/ft-2-OF_16/AGENT-NOTES.md`, `fts/ft-2-OF_17/AGENT-NOTES.md`, `fts/ft-2-OF_18/AGENT-NOTES.md`.

В таблицах описания свойств полей формы столбец `О` означает `Обязательность`, а столбец `Р` означает `Редактируемость`.

## Условный Контекст DaData

Если подтверждённый scope содержит поле, для которого ФТ прямо задаёт интеграцию с DaData, используй `work/vendor-references/dadata-reference.md` как внешний vendor reference.

- Приоритет источников: DOCX ФТ -> подтверждённые support/clarifications -> vendor reference -> наблюдение UI.
- Vendor reference описывает возможности DaData и помогает готовить шаги и UI-calibration, но не создаёт требования AutoFin.
- Исключение, подтверждённое пользователем для адресных полей: официальный контракт адресных подсказок DaData заменяет отдельный статический проектный справочник значений. Такую зависимость регистрируй как `external-dynamic-dictionary` с точным HTTPS/query binding; полный список регионов не выдумывай и не фиксируй как закрытый inventory.
- Компактная базовая модель: ввод пользователем -> список подсказок -> осознанный выбор варианта -> заполнение поля или связанных полей только в объёме, прямо заданном ФТ AutoFin.
- Не выводи из vendor reference настройки конкретной интеграции: trigger length, debounce, сортировку, fallback/retry, тексты UI-ошибок и набор автоматически заполняемых полей.
- Наблюдаемое поведение UI фиксируй отдельно. Расхождение UI с ФТ не превращай молча в новый expected result: это `mismatch-ft-ui`, дефект или coverage gap.
- В фазе `ft-ui-automation-prep` дополнительно используй `work/ui-automation-prep/UI-AGENT-NOTES.md`.
